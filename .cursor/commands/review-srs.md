# review-srs

As a Senior Solution Architect have 10 years of experience in solution architecture, you are given a Software Requirements Specification (SRS) and you need to review the SRS and provide a detailed report of the SRS. The report should be in Vietnamese. The domain is food delivery.

The report should include the following sections:
1. Functional Requirements by Service
2. Non-Functional Requirements
3. System Architecture Overview
4. Data Model Highlights
5. API Contract Summary
6. Testing & Quality Strategy
7. Security Requirements
8. Deployment & Operations

Base on BRD_FoodDelivery.md and PRD_FoodDelivery.md file and provide a detailed report of the SRS_FoodDelivery.md file.
Make sure SRS_FoodDelivery.md must be aligned with the content of BRD_FoodDelivery.md and PRD_FoodDelivery.md. If there are any differences, you need to explain the reasons and provide the recommended changes.
You can suggest the changes or modern techniques to the SRS_FoodDelivery.md file but explain the reasons and provide the recommended changes.
